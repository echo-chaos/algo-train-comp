## 毕业总结

### 为啥学的算法

```markdown
嗯，我本身不是计算机课班来着的，然后当时最开始毕业面试时常被问及算法相关的问题，结果也就不那么的理想，所以也开始去接触算法与数据结构相关的内容，不过大多数时候都是零零散散的学，并没有一个比较清晰的路径规划去完成里程碑式的学习。
```

### 训练营的过程

```markdown
一开始的几周课程相对之前有过接触会比较好接受些，随着课程的深入，相关的知识内容以及解题思考方式逐渐复杂，也开始不那么得心应手，很多时候一个知识点讲完还没回过神来。比如动态规划的状态转移方程，和图的一些思路并没有那么好反应得到。好在慢慢的养成来刷题的习惯，通过题目慢慢的培养对算法的熟练度。

对刷题的一些想法：
1、对一个题目审题题很重要很重要很重要
2、题目给出的边界条件要考虑的到，比如在期末考中的最后一题盖堡垒的题中，因为忽略了 0 < h[ i ] ≤ 2147483647 这个关键信息，导致明明感觉做法是ok的，但是提交测试用例只能通过一个，就一直停留在代码逻辑层面而耗费不少精力
3、题目中需要注意到一些过程答案是能够通过一些剪枝处理去进行优化的，就像棋中的四数之和一样，少了剪枝，最后的答案一直存在重复项
4、复杂度分析是属于对算法的一个很重要的理解，这一点做的一直不好，大多数时候都是做出来就好，很少回去分析到位复杂度情况，所以这一点我觉得需要养成
```
